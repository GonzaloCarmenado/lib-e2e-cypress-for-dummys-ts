## Blockers for 1.0.0

### B1 — [CRITICAL] No `LICENSE` file, but `package.json` declares MIT
`package.json` says `"license": "MIT"` yet no `LICENSE` file exists in the repo
root. Legally non-compliant and flagged by npm tooling.
**Fix:** add a root `LICENSE` (MIT text, author + year). npm auto-includes it.

### B2 — [CRITICAL] `export *` freezes internal symbols into the 1.0 contract
`src/index.ts:6-45` re-exports every module wholesale. This publishes into the
frozen public `.d.ts`:
- `_resetHttpMonitorState()` — explicitly `@internal` (`http-monitor.ts:256-257`), test-only.
- Singleton instances `translationService`, `transformationService`, `advancedTestTransformationService`.
- Internal constants `ACTIVE_SESSION_BREADCRUMB_KEY`, `RESUME_TTL_CONFIG_KEY`, `DEFAULT_RESUME_TTL_MINUTES`, `TOGGLE_MARGIN`, `DRAG_THRESHOLD`.
- Low-level helpers `makeSwalDraggableByContentId`, `setSwal2DataCyAttribute`, `ensurePopupDimensions`, `boxOffsetForDirection`, `clampTogglePosition`, `resolveExpandDirection`, `boxTopLeftFor`.

Every leaked name becomes a semver contract at 1.0.
**Fix:** replace `export *` with an explicit named-export allowlist of the intended
public surface; add `"stripInternal": true` to `tsconfig.json` so `@internal`
symbols are dropped from the `.d.ts`.

### B3 — [HIGH] Hardcoded Spanish strings bypass i18n (login-setup dialog)
`configuration.template.ts:316` (`Funciones detectadas`) and `:320`
(`Asignación de bloques`) are literal Spanish while every sibling uses `t(...)`.
Non-Spanish users see Spanish text. Violates the CLAUDE.md i18n rule.
**Fix:** add `CONFIG.LOGIN_SETUP_FUNCTIONS_DETECTED` + `CONFIG.LOGIN_SETUP_BLOCK_ASSIGN`
to all 5 language files, replace literals with `t(...)`.

### B4 — [MEDIUM→blocker] `configuration.ts` coverage 78.19% — below the 80% gate
Per CLAUDE.md the coverage gate is a hard rule and dropping below 80% requires
stopping. `configuration.ts` sits at 78.19% stmts/lines. Uncovered: issue-tracker
change handlers (`399-406,409`) and the login-setup save-button handler
(`433-443`). The global suite passes at 94%, but this file individually regressed.
**Fix:** add tests for the login-setup save path and issue-tracker handlers.

## High / medium findings (fix under this spec if cheap, else backlog)

### F1 — [MEDIUM] Redaction denylist is shallow — real credentials can leak to disk
`redact.utils.ts:1-9` only matches exact keys: `password`, `token`, `secret`,
`authorization`, `cookie`, `access_token`, `refresh_token`. It misses very common
variants: `apiKey`/`api_key`, `pwd`, `passwd`, `sessionId`, `idToken`/`id_token`,
`clientSecret`, `privateKey`, `pin`, `otp`, `ssn`, `cardNumber`. Any such field in
a captured JSON body is written verbatim into `cypress/fixtures/*.json` and
IndexedDB in cleartext.
**Fix:** match by substring/regex against a broader denylist (keys containing
`pass`, `secret`, `token`, `key`, `auth`, `cookie`, `credential`, `pin`, `otp`, `ssn`).

### F2 — [HIGH] README API signatures stale vs. actual `.d.ts`
README documents `insertTest(name, commands?, interceptors?, tags?)` but the real
signature is `insertTest(name, commands?, interceptors?, tags?, notes?, ticketId?)`.
Many public methods are undocumented (`hasDirectoryAccess`, `writeUploadedFile`,
`writeFixtures`, `saveLoginSetup`, `getLoginSetup`, recorder's `isVisible`,
`toggleVisibility()`, etc.).
**Fix:** regenerate README API blocks from current `.d.ts`; triage which methods
are intended public.

### F3 — [MEDIUM] No `CHANGELOG.md`
Only an ad-hoc "Breaking changes" note in the README. A 1.0.0 that freezes the API
should ship a changelog.
**Fix:** add `CHANGELOG.md` (Keep a Changelog), migrate the existing note, add 1.0.0 entry.

### F4 — [MEDIUM] Unvalidated login-setup feature would be frozen at 1.0
Per memory `spec_015_status`, login-setup was paused for security audit and lacks
user visual validation. `src/index.ts:3` still says "populated as migration phases
are completed".
**Fix:** either exclude login-setup persistence methods from the public surface
until validated, or mark them experimental; remove the migration framing.

### F5 — [MEDIUM] `lib-e2e-recorder.ts` remains a god-object (~970 lines)
Owns lifecycle, subscriptions, session continuity, widget-drag physics, 7 Swal
dialogs, save/export orchestration, recording-history. Largest change-risk surface.
**Fix (optional for 1.0):** extract `WidgetDragController`, `SessionContinuityController`,
`DialogManager`. Can be deferred to backlog — not a correctness issue.

### F6 — [MEDIUM] Duplicated magic-string storage keys
`'extendedHttpCommands'`, `'fixtureMode'` (×12 across files), `'e2e-recording-history'`
(×4) are raw literals; a typo in one silently breaks the sync cache. The codebase
already has the right pattern (`ACTIVE_SESSION_BREADCRUMB_KEY`).
**Fix:** hoist to exported constants.

## Low / info findings (backlog candidates)

- **[LOW] `assertion.utils.ts:19`** — selector not single-quote escaped; an
  `aria-label="it's here"` yields a broken JS string literal in generated tests.
  Fix: `escapeSingleQuotes(selector)` at the `cy.get()` boundary. *(Correctness bug — worth fixing.)*
- **[LOW] `recording.service.ts:161-164`** — interceptor URL wildcard embedded
  unescaped; a request path with `'` breaks the generated command. Fix: `escapeSingleQuotes(wildcard)`.
- **[LOW] `runner.ts:137`** — floating unhandled promise; a rejected `runSpec`
  leaves the HTTP response hanging. Fix: add `.catch` → 500. *(Runner only, Node target.)*
- **[LOW] `configuration.ts:379-399`** — six unguarded `as HTMLSelectElement` +
  `.addEventListener` with no null check; a template omission throws and the panel fails to mount.
- **[LOW] `escAttr` (`html.utils.ts:5`)** — does not escape `&`; non-canonical but
  attribute boundary is safe (quotes escaped). Fix: prepend `&`→`&amp;`.
- **[LOW] FSAA filenames** — `file.name` passed raw to `getFileHandle`; browser
  blocks traversal (throws on `/`,`\`,`.`,`..`) so no exploit, but a bad name aborts
  the whole write batch. Fix: sanitize basename + per-file try/catch.
- **[LOW] `advanced-test-editor.ts:116-129` & `persistence.service.ts:274-317`** —
  duplicated write/permission blocks. Fix: extract shared helpers.
- **[LOW] `file-preview.ts:92`** — floating clipboard promise (rejects on
  insecure context). Fix: `.catch(() => {})`.
- **[LOW] `lib-e2e-recorder.ts:526`** — vestigial no-op loop over interceptors in
  `recoverLastRecording()`; recovered interceptors are silently dropped. Decide: remove or implement.
- **[LOW] `filesystem-setup.ts`** coverage 5.88% — untested template + wiring.
- **[MEDIUM naming] `assertion-builder.ts` / `filesystem-setup.ts`** — HTML-generating
  helpers at component root that mix DOM access with markup; sidestep the pure-template
  rule via naming. Fix: split into pure `renderX` + `wireX`, or document as exception.
- **[INFO] `http-monitor.ts`** lacks `.service` suffix (naming inconsistency).
- **[INFO] package.json** missing `engines` and `publishConfig`; `author` email
  differs from git email (cosmetic).
- **[INFO] TSDoc** partial — several `private` members documented while core public
  methods (recorder `toggle`, `setLanguage`, services) are not.

## What's genuinely solid (evidence for confidence)

- **Escaping**: every `innerHTML =` renders through pure templates that escape all
  dynamic content (`escHtml`/`escAttr`); generated Cypress code escapes at the
  JS-string and CSS-attr layers at every emission site. `syntaxHighlight` escapes
  before wrapping. Two minor gaps (assertion selector, interceptor wildcard) noted above.
- **No header capture**: `http-monitor` reads only bodies, never
  Authorization/Cookie headers — the biggest token-leak surface is absent by design.
- **Redaction is wired before every sink** (fixtures + `cy.wait` validations) — the
  weakness is coverage of the denylist (F1), not the plumbing.
- **FSAA traversal blocked by the browser**; directory handles are opaque,
  permission-gated objects, not paths.
- **Runner not command-injectable**: `spawn` with `shell:false`, argv tokenized,
  binds `127.0.0.1` by default.
- **i18n parity**: 293 keys identical across es/en/fr/it/de.
- **Type safety**: 13 `as unknown as` all justified (FSAA/createElement/XHR-patch);
  zero `any`/`@ts-ignore`; non-null `!` elevated to lint error and code complies.
- **Concurrency**: AbortController wired on every listener + cleaned in `destroy()`;
  debounce timers keyed per element and flushed; http-monitor install/uninstall ref-counted.
